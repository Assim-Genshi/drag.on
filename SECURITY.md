# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.0 beta   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public GitHub issue.

Instead, report it privately by emailing assimgamre@gmail.com with:

A description of the vulnerability
Steps to reproduce the issue
The affected version (if known)
Any proof of concept or screenshots that may help
What to expect
Acknowledgement: Within 3 business days
Status updates: At least once a week while the report is being investigated
Resolution: If the vulnerability is accepted, I'll work on a fix as quickly as possible and keep you informed of the progress.

Once a fix is available, you'll be notified before the public disclosure whenever possible, and you'll be credited for the report if you'd like.

If the report is determined not to be a security vulnerability or cannot be reproduced, I'll explain why and may ask for additional information.

Thank you for helping keep this project and its users secure.
